CVS - Concurrent Versions System

CVS ist ein Sourcecode-Verwaltungstool für Entwicklergruppen. Arbeitsweise: CVS speichert eine zentrale Version des Programmcodes im sogenannten repository. Jeder Programmierer erhält eine eigene Kopie der Source-Dateien. Veränderungen des Quellcodes werden nur in der Kopie vorgenommen und nach der Fertigstellung mit CVS wieder den anderen Programmierern zur Verfügung gestellt. CVS vergibt dabei selbstständig Release-Nummern der Quellcodedateien.

Die wichtigsten Befehle:

commit

Syntax: cvs [cvs-options] commit [-lR][-m "log_text" | -F datei] [-r revision] [files...]
Vorausetzung: repository, Arbeitsverzeichnis
Effekte: Arbeitsverzeichnis
Synonyme: ci

Effekte:
Aktualisiert das repository mit den Änderungen der lokalen Kopie. Wird keine Datei als Parameter mitgegeben, wird das aktuelle Arbeitsverzeichnis samt Unterverzeichnisse als Paramter genommen. Existiert bereits eine neuere Version einer Datei im repository, bricht CVS den Befehl ab und meldet, das vorher ein update durchgeführt werden muß. Nach Beendigung des Befehls startet CVS den Standardeditor, damit ein Kommentar ins Logfile eingetragen wird. CVS erhöht nur die Revisionnummern von Dateien, die sich geändert haben (z. B. von 1.4 auf 1.5). Wichtig: den Befehl im Arbeitsverzeichnis ausführen.

commit options:

-l nur das aktuelle Verzeichnis als Parameter übergeben.
-R Verzeichnis samt Unterverzeichnisse als Parameter übergeben(default).
-r revision ändert alle Dateirevisionsnummern auf revisions (z.b. 3.0).
-m text trägt text ins Logfile ein ohne einen Editor zu starten.
-F datei trägt den Text aus datei ins Logfile ein, ohne einen Editor zu starten.

add

Syntax: cvs [cvs-options] add [-m test] files...
Vorausetzung: repository, Arbeitsverzeichnis
Effekte: -
Synonyme: -

Effekte
add markiert eine Datei im Arbeitsverzeichnis als zum Projekt gehörig. Sie wird aber erst durch den commit Befehl ins repository übernommen. Grundsätzlich werden neue Dateien mit der Revision x.1 versehen, wobei x für die höchste vorhandene Releasezahl steht.

add options:

-m text trägt text ins Logfile ein.

remove

Syntax: cvs [cvs-options] remove [remove-options] files...
Vorausetzung: repository, Arbeitsverzeichnis
Effekte: -
Synonyme: -

Effekte
remove markiert eine Datei im Arbeitsverzeichnis als im repository zu löschen. Sie wird aber erst durch den commit Befehl im repository gelöscht.

remove options:

-f führt anschließend den Befehl commit aus und löscht die Datei im repository.

update

Syntax: cvs [cvs-options] update [-dlP] [-d] files...
Vorausetzung: repository, Arbeitsverzeichnis
Effekte: Arbeitsverzeichnis
Synonyme: -

Effekte
Prüft das repository nach neuen Dateien/Datei-Revisionen und ändert entsprechend das Arbeitsverzeichnis.

update-options:
-m text fügt text ins Logfile ein.
-l nur das aktuelle Verzeichnis als Parameter übergeben
-P löscht leere Verzeichnisse
-d legt neue Verzeichnisse im Arbeitsverzeichnis an, falls Verzeichnisse im repository dazugekommen sind.
-I datei datei im Arbeitsverzeichnis wird beim update ignoriert. Die Option kann mehrfach verwendet werden.

checkout

Syntax: cvs [cvs-options] checkout [checkout-options] module...
Vorausetzung: repository
Effekte: Arbeitsverzeichnis
Synonyme: co, get

Effekte
Legt ein neues Arbeitsverzeichnis im aktuellen Verzeichnis an oder aktualisiert die Dateien im Arbeitsverzeichnis. Module sind Dateinamen, Verzeichnisse (mit Pfadangabe) oder eine Liste von beiden, definiert in der Datei modules. Zum Aktualisieren besser update verwenden (oder darauf achten, das checkout im Toplevel-Verzeichnis des Arbeitsverzeichnisses ausgeführt wird).

checkout options:
-l checkt nur das aktuelle Verzeichnis.
-P löscht leere Verzeichnisse.
-d dir beim Anlegen eines neuen Verzeichnisses wird dir statt module benutzt.

log

Syntax: cvs [cvs-options] log [log-options] [files...]
Vorausetzung: repository, Arbeitsverzeichnis
Effekte: -
Synonyme: -

Effekte
Gibt log-Informationen zu den Dateien aus.

log options:
- l nur log-Dateien des aktuellen Verzeichnisses werden ausgegeben.

diff

Syntax: cvs [cvs-options] diff [-I] [format_options] [[-r rev1] [-r rev2] [files...]
Vorausetzung: repository, Arbeitsverzeichnis
Effekte: -
Synonyme: -

Effekte
Vergleicht Dateien auf Unterschiede. Werden keine Dateinamen als Parameter mitgegeben, wird das aktuelle Verzeichnis samt Unterverzeichnisse als Parameter genommen.

diff options:
-l vergleicht nur Datieen im aktuellen Verzeichnis. -r rev1 -r rev2 vergleicht zwischen rev1 und rev2. format-options steuern die Ausgabe des Befehls. Die options ähneln GNU diff.

Javadoc

Das Tool javadoc generiert aus Java-Quellcode-Dateien automatisch eine Programm-Dokumentation im Stil der Java-API. Aus den Beschreibungen im Quelltext werden folgende Informationen zusammengetragen:

- Kopf
- Diagramm der Klassenhierarchie
- Klassenbeschreibung
- Index der Public-Variablen
- Index der Public-Methoden
- Index der Konstruktoren
- Beschreibung der Public-Variablen
- Beschreibung der Public-Konstruktoren
- Beschreibung der Public-Methoden

Javadoc-Kommentare haben die Form:

/**
*@...
* Kommentar
*/

In den Kommentaren können HTML-Tags verwendet werden, beispielsweise <i><i/>. Javadoc eigene Tags:

Javadoc-Tags

@see Klassenname
@see Klassenname#Methodenname
@see Ausgeschriebener Klassenname
@see Ausgeschriebener Klassenname#Methodenname
@version Versionstext
@author Author
@return Rückgabetext
@param Parametername Parametertext
@exception Exception-Klassenname Exceptiontext
@throws (siehe @exception)

Aufruf von javadoc:
javadoc sourcodefile.java
